# Book-My-Stay-App
Overview

This project demonstrates the first use case of the Hotel Booking Management System: establishing a clear and predictable starting point for the application. It shows how a basic Java program begins execution and produces console output.

This use case focuses on core Java concepts, object-oriented programming, and JavaDoc documentation, while keeping the implementation simple and console-based.

Objective

Show how a Java program executes from the main entry point.

Print a welcome message, application name, and version to the console.

Reinforce linear application flow and method invocation in Java.

Provide a foundation for later use cases involving bookings and data structures.

Actor

User: Runs the application from the command line or IDE.

Flow

User runs the application.

JVM invokes the main() method.

Application prints a welcome message along with the application name and version.

Application terminates successfully.

Key Concepts Demonstrated

Class: Encapsulates program behavior.

main() Method: Entry point for standalone Java applications.

static Keyword: Allows JVM to execute main() without creating an object.

Console Output: Using System.out.println() to print messages.

String Literals: Immutable text stored in the String pool.

Method Invocation: Demonstrates calling methods on objects (println() on System.out).

Application Flow: Linear, top-to-bottom execution.

JavaDoc Comments & Annotations: Documentation using @author and @version.
